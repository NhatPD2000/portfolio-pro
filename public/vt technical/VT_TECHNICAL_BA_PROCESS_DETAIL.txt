--------------------------
Thời gian: 2 giờ

  - CEO VT Technical

  - Giám đốc Kho

Nội dung:
  ✓ Giới thiệu dự án và mục tiêu

  ✓ Xác định stakeholders chính
  ✓ Lên lịch phỏng vấn

Kết quả:
  → Project Charter (5 trang)
  → Stakeholder List (15 người)
  → Interview Schedule (2 tuần)


Bước 1.2: Phỏng vấn Stakeholders
---------------------------------
Thời gian: 2 tuần (15 phỏng vấn x 1.5 giờ/phỏng vấn)

PHỎNG VẤN #1: Giám đốc Sales (Mr. Tuấn)
Thời gian: 1.5 giờ
Câu hỏi chính:
  Q1: Quy trình bán hàng hiện tại như thế nào?
  A1: "Hiện tại chúng tôi dùng Excel để quản lý khách hàng và đơn hàng. 
       Mỗi sales có file Excel riêng, không có hệ thống chung."
  
  Q2: Pain points lớn nhất là gì?
  A2: "Không biết được tình trạng đơn hàng real-time. Khách hàng hỏi 
       phải gọi điện cho kho, rất mất thời gian."
  
  Q3: Mong muốn gì từ hệ thống mới?
  A3: "Muốn có hệ thống tập trung, theo dõi được từ báo giá → đơn hàng 
       → giao hàng → thanh toán. Dashboard để xem doanh số real-time."

Pain Points được xác định:
  ❌ Dữ liệu rải rác trên nhiều file Excel
  ❌ Không theo dõi được trạng thái đơn hàng
  ❌ Mất thời gian tra cứu thông tin
  ❌ Không có báo cáo doanh số tự động
  ❌ Khó phối hợp với bộ phận kho

Requirements thu thập được:
  ✓ FR-SALES-001: Quản lý khách hàng tập trung
  ✓ FR-SALES-002: Quản lý báo giá
  ✓ FR-SALES-003: Quản lý đơn hàng với trạng thái
  ✓ FR-SALES-004: Dashboard doanh số real-time
  ✓ FR-SALES-005: Tích hợp với module kho


PHỎNG VẤN #2: Giám đốc Kho (Mr. Hùng)
Thời gian: 1.5 giờ
Câu hỏi chính:
  Q1: Quy trình quản lý kho hiện tại?
  A1: "Dùng sổ sách và Excel. Nhập/xuất kho ghi tay rồi nhập vào Excel 
       cuối ngày. Thường xuyên sai số."
  
  Q2: Vấn đề lớn nhất?
  A2: "Không biết tồn kho chính xác. Nhiều lần hứa giao hàng cho khách 
       nhưng kiểm tra kho thì không đủ. Mất uy tín."
  
  Q3: Yêu cầu với hệ thống mới?
  A3: "Cần theo dõi tồn kho real-time, cảnh báo khi hàng sắp hết, 
       quản lý nhiều kho (3 kho hiện tại)."

Pain Points:
  ❌ Tồn kho không chính xác (sai số 15%)
  ❌ Không có cảnh báo hàng sắp hết
  ❌ Nhập/xuất kho thủ công, dễ sai
  ❌ Không quản lý được nhiều kho
  ❌ Kiểm kê mất nhiều thời gian

Requirements:
  ✓ FR-INV-001: Quản lý tồn kho real-time
  ✓ FR-INV-002: Cảnh báo tồn kho min/max
  ✓ FR-INV-003: Quản lý nhiều kho
  ✓ FR-INV-004: Nhập/xuất kho tự động
  ✓ FR-INV-005: Kiểm kê định kỳ


PHỎNG VẤN #3: Giám đốc Dịch vụ (Ms. Lan)
Thời gian: 1.5 giờ
Pain Points:
  ❌ Không có hệ thống quản lý ticket
  ❌ Khách hàng gọi điện, không có lịch sử
  ❌ Không theo dõi được SLA
  ❌ Không có báo cáo dịch vụ

Requirements:
  ✓ FR-SVC-001: Ticket system
  ✓ FR-SVC-002: Quản lý hợp đồng dịch vụ
  ✓ FR-SVC-003: Lịch sử dịch vụ
  ✓ FR-SVC-004: SLA tracking
  ✓ FR-SVC-005: Báo cáo KPI dịch vụ

Bước 1.3: Phân tích Dữ liệu Lịch sử
------------------------------------
Thời gian: 1 tuần

Dữ liệu thu thập:
  • 2 năm dữ liệu bán hàng (2024-2025)
  • 2 năm dữ liệu kho
  • 2 năm dữ liệu dịch vụ

Phân tích:
  📊 Doanh số bán hàng:
     - 2024: 20 tỷ VND
     - 2025: 20.5 tỷ VND (tăng 2.5% - rất thấp)
     - Nguyên nhân: Mất khách do dịch vụ kém
  
  📊 Tồn kho:
     - Sai số trung bình: 15%
     - Hàng tồn kho lâu: 30% (6+ tháng)
     - Hàng thiếu: 20 lần/năm
  
  📊 Dịch vụ:
     - Thời gian xử lý ticket: 3-5 ngày
     - Tỷ lệ khách hàng hài lòng: 70%
     - Khách hàng rời bỏ: 15%/năm

Insight:
  💡 Vấn đề chính không phải là sản phẩm mà là quy trình
  💡 Cần cải thiện tốc độ xử lý và chất lượng dịch vụ
  💡 Tồn kho không hiệu quả làm tăng chi phí

Bước 2.3: Xây dựng Mô hình Dữ liệu
-----------------------------------
Thời gian: 3 ngày

ENTITIES CHÍNH:
┌──────────────────────────────────────────────────────────────┐
│ 1. Customer (Khách hàng)                                     │
│    - customer_id (PK)                                        │
│    - name, address, phone, email, tax_code                   │
│    - customer_type (VIP, Regular, Potential)                 │
│    - created_date, updated_date                              │
│                                                              │
│ 2. Product (Sản phẩm)                                        │
│    - product_id (PK)                                         │
│    - name, description, category, unit, price                │
│    - barcode, image                                          │
│    - created_date, updated_date                              │
│                                                              │
│ 3. Quotation (Báo giá)                                       │
│    - quotation_id (PK)                                       │
│    - customer_id (FK)                                        │
│    - quotation_date, valid_until                             │
│    - status (Draft, Sent, Approved, Rejected)                │
│    - total_amount, discount, tax, grand_total                │
│                                                              │
│ 4. Order (Đơn hàng)                                          │
│    - order_id (PK)                                           │
│    - customer_id (FK), quotation_id (FK)                     │
│    - order_date, delivery_date                               │
│    - status (Draft, Confirmed, Processing, Delivered)        │
│    - total_amount, discount, tax, grand_total                │
│                                                              │
│ 5. Warehouse (Kho)                                           │
│    - warehouse_id (PK)                                       │
│    - name, address, manager                                  │
│                                                              │
│ 6. Stock (Tồn kho)                                           │
│    - stock_id (PK)                                           │
│    - product_id (FK), warehouse_id (FK)                      │
│    - quantity, min_quantity, max_quantity                    │
│    - last_updated                                            │
│                                                              │
│ 7. Stock Movement (Nhập/Xuất kho)                           │
│    - movement_id (PK)                                        │
│    - product_id (FK), warehouse_id (FK)                      │
│    - movement_type (In, Out, Adjustment)                     │
│    - quantity, date, reference                               │
│                                                              │
│ 8. Service Contract (Hợp đồng dịch vụ)                      │
│    - contract_id (PK)                                        │
│    - customer_id (FK)                                        │
│    - start_date, end_date, status                            │
│                                                              │
│ 9. Ticket (Yêu cầu hỗ trợ)                                   │
│    - ticket_id (PK)                                          │
│    - customer_id (FK), contract_id (FK)                      │
│    - subject, description, priority                          │
│    - status (New, In Progress, Resolved, Closed)             │
│    - assigned_to, created_date, resolved_date                │
└──────────────────────────────────────────────────────────────┘

RELATIONSHIPS:
  Customer 1 ─── N Quotation
  Customer 1 ─── N Order
  Quotation 1 ─── 1 Order
  Order 1 ─── N Order_Line
  Product 1 ─── N Order_Line
  Product 1 ─── N Stock
  Warehouse 1 ─── N Stock
  Customer 1 ─── N Service_Contract
  Customer 1 ─── N Ticket
